# Phase 1.1: Capture Data Schemas

This document records the existing schemas for capture data across the extension and local service pipeline.

---

## 1. Data Collected by the Extension

The content script (`page-capture.ts`) extracts and stores capture data in `chrome.storage.local` under the key `surfrag:page-captures`.

### `PageCaptureRecord`

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique identifier: `${window.location.href}::${performance.timeOrigin}` (page URL + performance origin) |
| `title` | `string` | `document.title` or `"Untitled page"` |
| `url` | `string` | `window.location.href` |
| `referrer` | `string` | `document.referrer` or empty string |
| `bodyText` | `string` | Main body text extracted via Readability or `document.body.innerText`; max 100,000 chars |
| `maxScrollPercentage` | `number` | 0–100, rounded to 2 decimals |
| `createdAt` | `string` | ISO 8601 datetime when first captured |
| `updatedAt` | `string` | ISO 8601 datetime when last updated |

### TypeScript Definition (from `extension/surfrag-extension/types/capture.ts`)

```typescript
export type PageCaptureRecord = {
  id: string
  title: string
  url: string
  referrer: string
  bodyText: string
  maxScrollPercentage: number
  createdAt: string
  updatedAt: string
}
```

### Notes

- Records are upserted by `id`; scroll depth and body text may be updated on scroll/visibility change.
- History is capped at `MAX_CAPTURE_HISTORY` (100) entries.

---

## 2. Data Sent from Extension to Service

The extension sends captures to the local service via `POST /captures`. The payload is derived from `PageCaptureRecord` and does not include `id` (the service generates a new UUID).

### `CaptureSyncPayload`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `pageId` | `string` | Yes | Extension pageId (`${url}::${performance.timeOrigin}`) |
| `title` | `string` | Yes | Page title |
| `url` | `string` | Yes | Page URL |
| `referrer` | `string` | Yes | Referrer URL (may be empty) |
| `bodyText` | `string` | Yes | Main body text |
| `maxScrollPercentage` | `number` | Yes | 0–100 |
| `capturedAt` | `string` | Yes | ISO 8601 datetime (from `record.updatedAt`) |
| `sourceSession` | `string` | Yes | Session identifier (e.g. `"extension-content-script"`) |

### TypeScript Definition (from `extension/surfrag-extension/types/capture.ts`)

```typescript
export type CaptureSyncPayload = {
  pageId: string
  title: string
  url: string
  referrer: string
  bodyText: string
  maxScrollPercentage: number
  capturedAt: string
  sourceSession: string
}
```

### Service Validation (Zod `captureIngestSchema`)

The service validates the payload with these constraints:

| Field | Constraint |
|-------|------------|
| `pageId` | min 1, max 2500 chars |
| `title` | min 1, max 512 chars; whitespace collapsed |
| `url` | Valid URL, max 2048 chars |
| `referrer` | max 2048 chars; optional, defaults to `""` |
| `bodyText` | min 1, max 100,000 chars; whitespace collapsed |
| `maxScrollPercentage` | 0–100, default 0 |
| `capturedAt` | ISO 8601 datetime; optional (defaults to server time) |
| `sourceSession` | max 128 chars; optional, defaults to `""` |

## 3. Data Stored in SQLite

The local service persists captures in the `captures` table.

### Table: `captures`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `TEXT` | PRIMARY KEY | UUID generated by the service |
| `page_id` | `TEXT` | NOT NULL, UNIQUE | Extension pageId (`${url}::${performance.timeOrigin}`) |
| `title` | `TEXT` | NOT NULL | Page title |
| `url` | `TEXT` | NOT NULL | Page URL |
| `referrer` | `TEXT` | NOT NULL, DEFAULT `''` | Referrer URL |
| `body_text` | `TEXT` | NOT NULL | Main body text |
| `max_scroll_percentage` | `REAL` | NOT NULL | 0–100 |
| `captured_at` | `TEXT` | NOT NULL | ISO 8601 capture time |
| `source_session` | `TEXT` | NOT NULL, DEFAULT `''` | Session identifier |
| `created_at` | `TEXT` | NOT NULL | ISO 8601 when inserted |
| `updated_at` | `TEXT` | NOT NULL | ISO 8601 when inserted |

### Indexes

- `idx_captures_url` on `url`
- `idx_captures_captured_at` on `captured_at`
- `idx_captures_page_id` on `page_id`

### Drop Table (schema migration)

When the table schema changes, drop the old table before restarting the service:

```sql
DROP TABLE IF EXISTS captures;
```

Then restart the server; `CREATE TABLE IF NOT EXISTS` will create the new schema.

### DDL (from `services/local-mcp-server/src/db/sqlite.ts`)

```sql
CREATE TABLE IF NOT EXISTS captures (
  id TEXT PRIMARY KEY,
  page_id TEXT NOT NULL UNIQUE,
  title TEXT NOT NULL,
  url TEXT NOT NULL,
  referrer TEXT NOT NULL DEFAULT '',
  body_text TEXT NOT NULL,
  max_scroll_percentage REAL NOT NULL,
  captured_at TEXT NOT NULL,
  source_session TEXT NOT NULL DEFAULT '',
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_captures_url ON captures(url);
CREATE INDEX IF NOT EXISTS idx_captures_captured_at ON captures(captured_at);
CREATE INDEX IF NOT EXISTS idx_captures_page_id ON captures(page_id);
```

### Field Mapping (Extension → Service → SQLite)

| Extension (`PageCaptureRecord`) | Service (`CaptureRecord`) | SQLite (`captures`) |
|--------------------------------|---------------------------|---------------------|
| `id` | `pageId` | `page_id` |
| — | `id` (new UUID) | `id` (primary key) |
| `title` | `title` | `title` |
| `url` | `url` | `url` |
| `referrer` | `referrer` | `referrer` |
| `bodyText` | `bodyText` | `body_text` |
| `maxScrollPercentage` | `maxScrollPercentage` | `max_scroll_percentage` |
| `updatedAt` → `capturedAt` | `capturedAt` | `captured_at` |
| — (added at sync time) | `sourceSession` | `source_session` |
| `createdAt` (stored, not sent) | `createdAt` (server-generated) | `created_at` |
| `updatedAt` (sent as `capturedAt`) | `updatedAt` (server-generated) | `updated_at` |

**Notes:**
- **sourceSession**: Not stored in `PageCaptureRecord`. The content script adds it when building the sync payload (e.g. `"extension-content-script"`).
- **createdAt**: Extension stores it locally but does not send it. The service sets `created_at` from server time on insert.
- **updatedAt**: Extension sends it as `capturedAt`. The service sets `updated_at` from server time on insert.
